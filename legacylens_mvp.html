AI Summary
This is a legacy Java inventory management system built circa 2011, using JDBC for direct database access and Servlet-based HTTP handling. The application manages product catalog, stock levels, purchase orders, and user authentication across 12 classes.

The codebase uses a monolithic MVC pattern with tight coupling between business logic and data access. Database connections are managed manually without connection pooling, creating scalability risks under load.

Key capabilities: User auth & RBAC, inventory CRUD, PO workflow, basic reporting via raw SQL, session management via HttpSession.

Migration Roadmap
1
Security Hardening & Critical Fixes
⏱️ 2 weeks  |  💰 ₹1,20,000  |  👥 2 developers
Immediately fix SQL injection via PreparedStatements, move credentials to environment variables/vault, implement bcrypt password hashing. These are P0 security fixes that can be done without restructuring the app.
2
Refactor Data Access Layer
⏱️ 3 weeks  |  💰 ₹2,25,000  |  👥 3 developers
Replace raw JDBC with Spring Data JPA + HikariCP connection pooling. Introduce proper repository pattern. Fix N+1 queries with JOIN fetch queries. Add proper transaction management.
3
Migrate to Spring Boot 3
⏱️ 4 weeks  |  💰 ₹3,00,000  |  👥 3 developers
Migrate from Servlet/Tomcat 7 to Spring Boot 3 + embedded Tomcat 10. Replace JSP views with REST APIs (for frontend separation). Add Spring Security for CSRF, session management, and auth.
4
Testing, Docs & Deployment
⏱️ 3 weeks  |  💰 ₹1,75,000  |  👥 2 developers
Write unit + integration tests (JUnit 5, Mockito, Testcontainers). Generate API documentation (OpenAPI/Swagger). Set up CI/CD pipeline (GitHub Actions). Deploy to AWS/Azure with containerization (Docker).
